# Technolnstrument — каталог промышленных инструментов

Сайт-каталог по продаже промышленных инструментов. Реализован на **Django** (Python). Структура и UX ориентированы на каталог Krona: хедер, подкатегории, боковые фильтры, сортировка, сетка карточек товаров.

## Запуск (Django)

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Запустите сервер разработки:

**Вариант А** — дважды кликнуть `run-django.bat` (подставит путь к Python сам).

**Вариант Б** — в PowerShell из папки проекта:

```powershell
# Если Python не в PATH — укажите полный путь (подставьте свою версию: Python313, Python312 и т.д.):
& "$env:LOCALAPPDATA\Programs\Python\Python313\python.exe" manage.py runserver
```

Если `py` или `python` есть в PATH: `py manage.py runserver` или `python manage.py runserver`.

3. Откройте в браузере: **http://127.0.0.1:8000/**

## База данных (SQLite)

Используется SQLite (`db.sqlite3` в корне проекта). Таблицы:

- **Товары:** категории (`Category`), товары (`Product` — название, цена, бренд, категория, изображение, наличие).
- **Пользователи:** стандартная модель Django `User` (логин, email, пароль) — для регистрации и входа.
- **История заказов:** заказы (`Order` — пользователь, контакты, адрес, статус, сумма), позиции заказа (`OrderItem` — товар, количество, цена на момент заказа).
- **Бухгалтерия:** платежи (`Payment` — по заказу, сумма, способ оплаты, статус), проводки (`AccountingEntry` — приход/расход, дата, сумма, описание, связь с заказом/платежом).

Первый вход в админку — создайте суперпользователя (в PowerShell из папки проекта):

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python313\python.exe" manage.py createsuperuser
```

Если в PATH есть `py`: `py manage.py createsuperuser`.

Затем откройте **http://127.0.0.1:8000/admin/** и войдите. В админке доступны: пользователи (Users), категории, товары, заказы, платежи, бухгалтерия (проводки).

## Структура проекта

- `config/` — настройки Django (settings, urls, wsgi)
- `main/` — приложение: главная страница, модели, админка
  - `models.py` — Category, Product, Order, OrderItem, Payment, AccountingEntry
  - `admin.py` — регистрация моделей в админ-панели
  - `templates/main/index.html` — шаблон каталога
  - `static/main/` — CSS, JS
- `manage.py` — точка входа Django
- `db.sqlite3` — база SQLite (создаётся после `migrate`)
